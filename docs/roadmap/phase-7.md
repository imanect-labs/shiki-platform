# Phase 7 — 資料作成 v1（ライブラリ直接生成）

> 目的: パワポ/ワード/エクセルを**ライブラリ直接生成**でプログラム的に作る縦スライスを完成させる。
> xlsx は Rust（`rust_xlsxwriter`）、docx/pptx は ingestion-worker（Python: python-docx / python-pptx）で生成し、
> 既存ひな型の**プレースホルダ穴埋め方式**を併設する（エンプラで実用的・事故が少ない）。生成物は StorageService に保存され、
> 書込イベント経由で**自動RAG再索引**される。チャット／サンドボックスのエージェント（Phase 5）が「スペック→生成→保存」で生成できる。
> Euro-Office/LibreOffice は**PDF変換/プレビュー用途に限定**。ブラウザ内編集（OnlyOffice/Collabora）は**v2＝並行/将来トラック**で本フェーズ対象外。
> 完了の定義(DoD): ユーザー（またはエージェント）がスペック／ひな型を指定すると、docx/pptx/xlsx が生成され、
> StorageService に権限付きで保存され、RAG に再索引され、必要に応じて PDF プレビューが見られる。

## 詳細

### Task 7.1: `DocumentGenerator` トレイト＋生成ジョブ基盤
- **area**: docgen / **path**: `crates/docgen`
- **依存**: 3.5, 1.8
- **仕様**:
  - `DocumentGenerator` トレイト: `generate(spec) -> DocumentArtifact`。フォーマット（docx/pptx/xlsx）ごとに実装を差せる。
  - **スペック**は構造化（中間表現: ドキュメント種別＋セクション/スライド/シート＋データ）。
    生成系の選択（Rustネイティブ vs Python worker）はトレイト解決で隠蔽。
  - 生成は**ジョブ**として実行（同期生成＋大物は pgmq 投入）。生成結果は **StorageService に保存**（Task 7.6）するのが終端。
- **受け入れ条件**:
  - [ ] 同一スペックから種別に応じた生成実装にディスパッチできる
  - [ ] 生成成功でストレージnodeへの参照を含む `DocumentArtifact` が返る
  - [ ] 生成失敗が監査/トレースに残る

### Task 7.2: xlsx 生成（`rust_xlsxwriter`）
- **area**: docgen / **path**: `crates/docgen`
- **依存**: 7.1
- **仕様**:
  - `rust_xlsxwriter` で xlsx を**Rustネイティブ生成**（外部プロセス不要）。シート/セル/書式/数式/簡易チャートをスペックから組む。
  - 大きな表データのストリーミング書込に対応（メモリ上限を守る）。
- **受け入れ条件**:
  - [ ] スペックから複数シート・書式付き xlsx が生成される
  - [ ] 数式と基本書式（数値/日付/スタイル）が反映される
  - [ ] 生成物が StorageService に保存される

### Task 7.3: docx/pptx 生成（ingestion-worker: python-docx / python-pptx）
- **area**: docgen / **path**: `ingestion-worker/`, `crates/docgen`
- **依存**: 7.1, 2.8
- **仕様**:
  - docx は `python-docx`、pptx は `python-pptx` で生成。**Phase 2 の ingestion-worker（Python）に生成エンドポイント／ジョブを追加**（既存基盤の応用）。
  - Rust 側 `DocumentGenerator` 実装は worker を RPC/ジョブ投入で呼び、生成物バイト列を受け取りストレージ保存へ渡す。
  - スペック→見出し/段落/表（docx）、スライド/レイアウト/テキストボックス/図表（pptx）へマッピング。
- **受け入れ条件**:
  - [ ] スペックから段落・表を含む docx が生成される
  - [ ] スペックから複数スライドの pptx が生成される
  - [ ] worker 生成物が Rust 側経由でストレージ保存される

### Task 7.4: ひな型プレースホルダ穴埋め方式
- **area**: docgen / **path**: `crates/docgen`, `ingestion-worker/`
- **依存**: 7.2, 7.3
- **仕様**:
  - 既存の docx/pptx/xlsx **ひな型ファイルのプレースホルダ（`{{key}}` 等）を値で置換**する穴埋め生成を併設。
    レイアウト/ブランドを崩さず差し込めるため**エンプラで実用的・事故が少ない**。
  - 表/リストの**行リピート**、画像差し込み、条件付きセクションをサポート。docx/pptx は worker、xlsx は Rust ネイティブで穴埋め。
  - 必須プレースホルダ未充足はエラー（穴あき生成を防止）。
- **受け入れ条件**:
  - [ ] ひな型のプレースホルダがデータで置換され元のレイアウトを保つ
  - [ ] 表/リストの行リピートと画像差し込みができる
  - [ ] 未充足の必須プレースホルダで生成が失敗する

### Task 7.5: テンプレ管理（ひな型の登録/バージョン/共有）
- **area**: docgen / **path**: `crates/docgen`, `crates/storage`, `crates/authz`
- **依存**: 7.4, 1.6
- **仕様**:
  - ひな型を**バージョン付きアーティファクト**として登録（実体は StorageService、メタは docgen）。プレースホルダのスキーマ（必須キー/型）を保持。
  - **ReBAC でロール/個人に共有**（既存の共有可能アーティファクト枠に乗せる）。差し替え時は新バージョン。
- **受け入れ条件**:
  - [ ] ひな型を登録・更新（バージョン）できる
  - [ ] ひな型を個人/ロールに ReBAC で共有・解除できる
  - [ ] テンプレのプレースホルダスキーマを参照して穴埋めを検証できる

### Task 7.6: 生成物のストレージ保存＋RAG再索引フック
- **area**: docgen / **path**: `crates/docgen`, `crates/storage`, `crates/rag`
- **依存**: 7.1, 2.9
- **仕様**:
  - 全生成物を **StorageService に保存**（権限・バージョン・コンテンツアドレッシングは既存経路）。保存で**書込イベントが発行**される。
  - 書込イベント→ジョブキュー→ingestion-worker の経路（Task 2.9）に乗り、生成した docx/pptx/xlsx が**自動でRAG再索引**される。
  - 保存先フォルダ/権限は呼び出しユーザー（またはエージェント）の権限に従う。
- **受け入れ条件**:
  - [ ] 生成物が権限付きでストレージに保存される
  - [ ] 保存で書込イベントが発行され RAG に再索引される
  - [ ] 再索引後、生成文書が permission-aware 検索でヒットする

### Task 7.7: 生成ツール（チャット/エージェントからの呼出）
- **area**: agent / **path**: `crates/agent-core`, `crates/docgen`
- **依存**: 7.2, 7.3, 7.4, 3.9
- **仕様**:
  - `generate_document(format, spec | template_id + values, dest)` を **`Tool`** として提供。チャット（agent-core 制約版）と
    **サンドボックスのエージェント（Phase 5）の「スペック→生成コード実行→ストレージ保存」フロー**の両方から呼べる。
  - **生成は破壊系ではないが書込を伴う**ため、Task 3.9 の保存先権限/明示許可ポリシに従う。生成結果（ストレージ参照）を content block / ツール結果で返す。
- **受け入れ条件**:
  - [ ] チャットで「〜の資料を作って」からツールが起動し生成物が保存される
  - [ ] Phase 5 エージェントが生成ツール経由でストレージに資料を残せる
  - [ ] 保存先の権限がツール呼び出しユーザー/エージェントに従う

### Task 7.8: PDF変換（LibreOffice/Euro-Office, 変換用途限定）
- **area**: docgen / **path**: `crates/docgen`, `deploy/`
- **依存**: 7.6
- **仕様**:
  - **LibreOffice（headless）/ Euro-Office を変換用途に限定**して docx/pptx/xlsx → PDF を変換（プレビュー/PDF出力が要る時のみ）。
    編集には使わない（ブラウザ内編集は v2）。
  - 変換はサンドボックス的に隔離した変換プロセス／コンテナで実行。生成PDFもストレージ保存（任意で再索引）。
- **受け入れ条件**:
  - [ ] docx/pptx/xlsx を PDF に変換できる
  - [ ] 変換プロセスが隔離環境で実行される
  - [ ] 変換は変換用途のみで編集機能を持たない

### Task 7.9: 生成物プレビュー＋生成UI
- **area**: frontend / **path**: `web/`
- **依存**: 7.7, 7.8
- **仕様**:
  - 生成UI: テンプレ選択（共有されたひな型一覧）／スペック・穴埋め値入力／フォーマット選択→生成実行。
  - 生成後の**プレビュー**（PDF変換結果の表示）と、ストレージ上の生成物へのリンク／ダウンロード。チャット内からの生成結果もプレビュー可能。
  - **ブラウザ内編集はここでは提供しない**（v2＝将来トラック）。
- **受け入れ条件**:
  - [ ] テンプレ選択＋値入力から資料を生成できる
  - [ ] 生成物の PDF プレビューが表示される
  - [ ] 生成物がストレージから開ける/ダウンロードできる
